**Prompt 1 — Architecture générale d'un système de Business Intelligence**

```
Créer un diagramme d'architecture montrant les composants d'un système de Business Intelligence complet.

Disposition en flux horizontal de gauche à droite avec 4 blocs principaux reliés par des flèches :

Bloc 1 - SOURCES DE DONNÉES (à gauche) :
- Icône de fichier : Fichiers (CSV, Excel, JSON, XML)
- Icône de base de données : Bases de données (PostgreSQL, MySQL, Oracle)
- Icône de cloud : API REST / Services Web
- Icône de tableau : Fichiers plats (TSV, logs)

Bloc 2 - PROCESSUS ETL (au centre-gauche) :
- Extraction : connecteurs vers les sources
- Transformation : nettoyage, standardisation, agrégation
- Chargement : insertion dans l'entrepôt

Bloc 3 - DATA WAREHOUSE (au centre-droit) :
- Entrepôt de données centralisé
- Schéma en étoile (table de faits + dimensions)
- Données historisées et optimisées pour l'analyse

Bloc 4 - COUCHE DE PRÉSENTATION (à droite) :
- Tableaux de bord interactifs
- Graphiques (barres, lignes, circulaires)
- Indicateurs KPI
- Rapports et exports

Flèches entre chaque bloc montrant le flux de données : Sources → ETL → Data Warehouse → Visualisation.

Style professionnel, fond blanc, couleurs distinctes pour chaque bloc (bleu pour sources, orange pour ETL, vert pour Data Warehouse, violet pour présentation), texte en français.
```

**Prompt 3 — Comparaison OLTP vs OLAP**

```
Créer un diagramme comparatif entre les bases de données OLTP et OLAP (Data Warehouse).

Disposition en deux colonnes côte à côte avec un titre pour chaque :

COLONNE GAUCHE - OLTP (Online Transaction Processing) :
Couleur : bleu clair
- Objectif : Transactions quotidiennes
- Opérations : INSERT, UPDATE, DELETE fréquents
- Données : Actuelles, en temps réel
- Requêtes : Simples, courtes, sur peu de lignes
- Utilisateurs : Employés opérationnels
- Structure : Tables normalisées (3NF)
- Volume : Gigaoctets
- Exemple : Système de gestion de commandes

COLONNE DROITE - OLAP (Online Analytical Processing) :
Couleur : vert
- Objectif : Analyse et aide à la décision
- Opérations : SELECT complexes, agrégations
- Données : Historiques, consolidées
- Requêtes : Complexes, longues, sur millions de lignes
- Utilisateurs : Analystes, décideurs
- Structure : Schéma en étoile (dénormalisé)
- Volume : Téraoctets
- Exemple : Data Warehouse pour l'analyse des ventes

Au centre, une flèche bidirectionnelle avec le texte "ETL" montrant que les données passent de l'OLTP vers l'OLAP.

Style professionnel sous forme de tableau comparatif, fond blanc, texte en français.
```
